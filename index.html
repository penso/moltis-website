Moltis

A personal AI gateway, built in Rust.

~ quickstart

$ curl -fsSL https://www.moltis.org/install.sh | sh

$ brew install moltis-org/tap/moltis

# Pre-built binary via cargo-binstall (no compilation)

$ cargo binstall moltis

# Multi-arch: amd64/arm64

$ docker pull ghcr.io/moltis-org/moltis:latest

# Build from source

$ cargo install moltis --git https://github.com/moltis-org/moltis

# Download packages from GitHub Releases

# Debian/Ubuntu (.deb)

$ sudo dpkg -i moltis_*_amd64.deb

# Fedora/RHEL (.rpm)

$ sudo dnf install moltis-*.x86_64.rpm

# Arch Linux (.pkg.tar.zst)

$ sudo pacman -U moltis-*-x86_64.pkg.tar.zst

# Snap

$ sudo snap install moltis

# AppImage

$ ./moltis-*-x86_64.AppImage

Single Binary

One file runs your entire AI gateway. No Node.js, no npm, no runtime dependencies to babysit.

Memory Safe

Secrets are zeroed on drop, not "eventually collected." No unsafe code anywhere in the codebase.

Authentication

HTTPS by default. Password, token, and passkeys access — secure from the first request.

Compile-time Safe

Provider misconfigurations are caught by the compiler, not discovered at 2 AM in production.

SSRF Protected

Web fetch blocks loopback, private, and link-local IPs. The LLM can't reach your internal services.

Streaming-first

Responses appear the moment the first token arrives. Lock-free circuit breakers, no GC latency spikes.

Skills & MCP

Extend with plugins, skills, and MCP tool servers. Stdio or HTTP/SSE transports, auto-restart on crash.

FS or Sandbox

Full filesystem access for trusted work, or per-session Docker/Apple Container isolation. Your call.

Long-term Memory

Embeddings-powered knowledge base with hybrid vector + full-text search. Your agent remembers context.

Multi-LLM

OpenAI, Anthropic, Google, and 30+ more providers — or run fully local. Bring your own model.

Multi-Channel

Chat from the web UI, Telegram, or API. One agent, multiple frontends — responses sync everywhere.

Trusted Network

Domain-filtering proxy for sandbox containers. Allowlist sites or approve requests interactively.
